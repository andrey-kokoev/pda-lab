# build-update-visibility

Benchmark package for a real `telesense` thread where a seemingly simple status feature contained hidden delegation and product-state ambiguity.

## Ambiguity Type

Delegation/task-shaping.

## Package Shape

- `raw/task.md` original messy task entry point
- `reference/closed-formulation.md` reference de-arbitrarized formulation
- `reference/expected-policy-questions.md` expected ambiguity-clearing questions
- `validation/criteria.md` execution validation criteria
- `baseline.md` repo-state reference for the benchmark
