# Chosen Directions

These benchmark-specific directions are already settled and should be preserved in generated handoffs rather than reopened or replaced with generic defaults.

- The problem is build/update-state legibility, not deploy timestamp display.
- The relevant visible state is: current build identity, latest available build identity, and whether refresh is needed.
- Placement is landing-view only.
- Call-view or in-session UI is out of scope.
- Presentation should be low-weight and non-modal.
- Deploy timestamp may appear only as secondary metadata, not the primary signal.
