# Reference Closed Formulation

Target closure for this benchmark:

- The build/update state element is secondary status information, not a primary control.
- The core ambiguity is placement and presentation weight, not data-model semantics.
- The status should not read as modal, banner-like, or interruptive.
- Scope should be narrowed to the landing view rather than call surfaces.
- Recomposition is allowed only once placement, visual weight, and surface scope are explicit.

The benchmark should reward traces that reduce UI overstatement and clarify where status belongs before implementation.
