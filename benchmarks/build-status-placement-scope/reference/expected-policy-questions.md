# Expected Policy Questions

A strong PDA trace should make some version of these decision points explicit:

1. Is this UI element primary action or secondary status?
2. Is the current presentation too modal or overstated for its role?
3. Where should the status live in the information hierarchy?
4. Should it appear only on landing view, or also on call surfaces?
5. Should the element be a badge/widget, or plain muted text?
